# Video ReEncoder - 视频批量压缩工具

一个基于 Python 和 FFmpeg 的视频批量压缩工具，支持多种视频格式，可自定义输出码率和目录结构。

## 功能特性

- ✅ **批量处理**：支持批量压缩多个视频文件
- ✅ **多格式支持**：支持 MP4、MKV、AVI、MOV、FLV、WebM、WMV、M4V 等主流视频格式
- ✅ **递归处理**：可选择是否递归处理子目录中的视频文件
- ✅ **灵活输出**：支持自定义输出目录或保存在源文件同目录（自动添加 `_compressed` 后缀）
- ✅ **码率可调**：自由设置目标视频码率（如 500K、1M、2M 等）
- ✅ **音频保持**：自动检测并保持原始音频码率
- ✅ **进度显示**：实时显示处理进度和成功/失败统计

## 系统要求

- Python 3.7+
- FFmpeg（需要预先安装并配置到系统环境变量）
- Windows / Linux / macOS

## 安装依赖

```
bash
pip install -r requirements.txt
```
**依赖说明：**
- `ffmpeg-python>=0.2.0`：Python 操作的 FFmpeg 绑定库

> ⚠️ **重要提示**：除了安装 `ffmpeg-python` 库外，还需要单独安装 FFmpeg 二进制文件：
> - **Windows**: 从 [ffmpeg.org](https://ffmpeg.org/download.html) 下载并添加到 PATH
> - **Linux**: `sudo apt-get install ffmpeg` (Debian/Ubuntu) 或 `sudo yum install ffmpeg` (CentOS)
> - **macOS**: `brew install ffmpeg`

## 使用方法

### 基本语法

```
bash
python main.py -i <输入目录> [-o <输出目录>] [-b <目标码率>] [-r]
```
### 参数说明

| 参数 | 简写 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--input-dir` | `-i` | 是 | - | 输入视频目录路径 |
| `--output-dir` | `-o` | 否 | 源文件同目录 | 输出目录路径 |
| `--bitrate` | `-b` | 否 | 1000K | 目标视频码率（如：500K, 1M, 2M） |
| `--recursive` | `-r` | 否 | False | 是否递归处理子目录 |

### 使用示例

#### 示例 1：基本使用（默认码率 1000K）
```
bash
python main.py -i ./videos
```
输出文件将保存在源文件同目录，文件名添加 `_compressed` 后缀。

#### 示例 2：自定义码率和输出目录
```
bash
python main.py -i ./videos -o ./output -b 2M
```
将视频压缩为 2M 码率，并保存到 output 目录。

#### 示例 3：递归处理子目录
```
bash
python main.py -i ./videos -o ./output -b 1500K -r
```
递归处理 videos 目录下所有子目录中的视频文件。

#### 示例 4：指定更低码率
```
bash
python main.py -i ./videos -b 500K
```
压缩为 500K 码率，适合网络传输。

## 技术细节

### 视频编码参数
- **视频编码器**：H.264 (libx264)
- **音频编码器**：AAC
- **音频码率**：自动检测并保持原始音频码率（如果检测失败则使用 128K）

### 工作流程

1. **扫描视频文件**：根据指定目录和扩展名查找所有视频文件
2. **检测音频码率**：使用 FFmpeg probe 分析视频文件的音频流
3. **转码处理**：使用 H.264 重新编码视频，保持原始分辨率和音频码率
4. **临时文件保护**：先输出为临时文件，完成后重命名，避免损坏原文件
5. **批量统计**：显示处理进度和成功/失败统计

### 支持的码率格式

- `500K` - 低码率，适合移动端
- `1000K` - 中等码率，平衡画质和体积（默认）
- `1500K` - 较高码率
- `2M` / `2000K` - 高码率，接近原画质
- `4M` / `4000K` - 超高清

## 注意事项

1. **FFmpeg 依赖**：确保 FFmpeg 已正确安装并可在命令行中访问
2. **磁盘空间**：转码过程会生成临时文件，请确保有足够的磁盘空间
3. **原文件保护**：程序不会修改原始文件，输出文件会有不同的名称或位置
4. **编码时间**：视频转码需要较长时间，请耐心等待
5. **错误处理**：如果某个文件编码失败，会继续处理下一个文件

## 常见问题

### Q: 为什么安装 ffmpeg-python 后仍然报错？
A: ffmpeg-python 只是 Python 的绑定库，还需要单独安装 FFmpeg 二进制文件并配置到系统 PATH。

### Q: 如何调整视频分辨率？
A: 当前版本保持原始分辨率。如需调整分辨率，可以修改 `encode_video` 方法中的 FFmpeg 参数，添加 `scale` 滤镜。

### Q: 支持哪些音频编码格式？
A: 输出统一使用 AAC 编码。如果原视频没有音频流，程序也能正常处理。
